# 新メンバー向け開発環境セットアップ

## 必要なソフトウェア

### 1. Node.js
- **バージョン**: 18.0以上
- **ダウンロード**: https://nodejs.org/
- **確認方法**: `node --version`

### 2. Git
- **ダウンロード**: https://git-scm.com/
- **確認方法**: `git --version`

### 3. エディタ（推奨）
- **VS Code**: https://code.visualstudio.com/
- **推奨拡張機能**:
  - Vue Language Features (Volar)
  - ESLint
  - Prettier
  - GitLens

## セットアップ手順

### 1. リポジトリをクローン
```bash
git clone https://github.com/haguro195-commits/election-site-2025.git
cd election-site-2025
```

### 2. 依存関係をインストール
```bash
npm install
```

### 3. 開発サーバーを起動
```bash
npm run dev
```

### 4. ブラウザで確認
- http://localhost:3000 にアクセス
- サイトが正常に表示されることを確認

## 初回作業

### 1. Gitの設定
```bash
git config --global user.name "あなたの名前"
git config --global user.email "あなたのメール"
```

### 2. 新しいブランチを作成
```bash
git checkout -b feature/あなたの名前-初回作業
```

### 3. 簡単な変更を加える
- README.mdに自分の名前を追加
- または新しいコンポーネントを作成

### 4. 変更をコミット・プッシュ
```bash
git add .
git commit -m "feat: 新メンバー参加 - 初回セットアップ完了"
git push origin feature/あなたの名前-初回作業
```

### 5. プルリクエストを作成
- GitHubでPull Requestを作成
- レビューを依頼

## 開発ルール

### ブランチ命名規則
- `feature/機能名` - 新機能開発
- `fix/修正内容` - バグ修正
- `docs/文書名` - ドキュメント更新

### コミットメッセージ
- `feat: 新機能の説明`
- `fix: バグ修正の説明`
- `docs: ドキュメント更新の説明`

## 質問・相談

### 困った時の連絡方法
1. **GitHub Issues**: バグ報告・機能要望
2. **GitHub Discussions**: 技術相談
3. **直接連絡**: メール・チャット

### よくある質問
- **Q**: npm installでエラーが出る
- **A**: Node.jsのバージョンを確認してください

- **Q**: 開発サーバーが起動しない
- **A**: ポート3000が使用中の可能性があります

- **Q**: Gitでプッシュできない
- **A**: 最新のmainブランチを取得してください
